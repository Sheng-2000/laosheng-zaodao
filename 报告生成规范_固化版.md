# 「老盛早知道」报告生成质量保障规范（固化版）

## 一、前置检查清单（生成前必做）

| 检查项 | 要求 | 验证方式 |
|--------|------|----------|
| 模板文件存在 | `template.html` 必须存在 | `ls template.html` |
| 模板完整性 | 3322行，223个CSS类 | `wc -l` + `grep -c 'class='` |
| 旧报告参考 | 读取昨日报告确认日期衔接 | 检查期号连续性 |

## 二、生成流程（强制顺序）

```
步骤1: cp template.html 老盛早知道_YYYYMMDD.html
       └── 严禁直接写新文件，必须复制模板

步骤2: 数据搜索（最多4轮，每轮5个，总计≤20次）
       ├── 第1轮：A股收盘 + 美股隔夜 + 国内外新闻 + 央行动态
       ├── 第2轮：港股/亚太/欧洲 + 大宗商品 + 加密货币 + 汇率国债
       ├── 第3轮：AI前沿 + 关注标的 + 券商策略 + 理财数据
       └── 第4轮：补充缺失 + 交叉验证

步骤3: 内容替换（按区域逐个替换）
       ├── Header（期号/日期/ticker/标题）
       ├── Tab 0: 要点速览（4卡片+8要点+时间线）
       ├── Tab 1: 国内外新闻（8张卡片）
       ├── Tab 2: AI前沿（5个section）
       ├── Tab 3: 全球市场（8个market-block，内容要丰富）
       ├── Tab 4: 价值投资风向（7个sub-title）
       ├── Tab 5: 关注标的（13只标的+深度解读）
       ├── Tab 6: 理财话题（8个sub-title）
       ├── Tab 7: 今日总结（7个模块）
       └── Footer（数据截至日期）

步骤4: 质量检查（15项清单逐项勾选）
步骤5: 更新index.html的reports数组
步骤6: bash sync_github.sh "提交信息"
```

### 3.2 数据准确性要求

| 数据类型 | 要求 | 禁止 |
|----------|------|------|
| 股价 | 使用搜索到的真实收盘价 | ❌ 编造股价 |
| 涨跌幅 | 与股价匹配，涨红跌绿 | ❌ 颜色错误 |
| 成交额 | 标注单位（亿/万亿） | ❌ 单位缺失 |
| 日期 | 明确标注数据日期 | ❌ 模糊表述 |
| 股息率 | 区分A股/H股，搜索验证 | ❌ 编造股息率 |

### 3.3 原创性要求

| 检查项 | 标准 |
|--------|------|
| 与昨日报告对比 | 关键段落连续相同字数 ≤ 30字 |
| 日期相关表述 | 必须更新（如"今日""明日"） |
| 数据驱动分析 | 基于当日数据重新撰写 |
| 禁止 | 同义词替换、句式调整等机械改写 |

## 四、15项质量检查清单（生成后必检）

```
□ 1. 价值投资风向完整结构（7个子板块）
□ 2. 社区情绪9条（按固定顺序：高股息→银行→公用事业→电力→AI算力→能源油价→新能源→黄金→加密货币）
□ 3. 估值表9行3列
□ 4. 关注标的13个（银行7+公用事业2+央企4）

□ 9. 要点速览（4条提示条+8个highlight+日历）
□ 10. 全球市场8个板块（BTC+ETH，零"待更新"）
□ 11. 数据准确性+涨跌颜色（涨红跌绿）

```

## 六、禁止事项（红线）

| ❌ 禁止行为 | 后果 |
|-------------|------|
| 不复制模板直接写HTML | 结构缺失、CSS类丢失 |
| 修改template.html的CSS | 样式不一致 |
| 编造股价/股息率/数据 | 数据错误、信任危机 |
| 复制昨日报告做同义词替换 | 原创性检查失败 |
| 省略任何sub-section/card/grid | 结构不完整 |
| 超过20次搜索 | 效率低下、资源浪费 |
| 不检查15项清单直接推送 | 质量问题漏检 |
