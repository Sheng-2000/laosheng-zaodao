## 一、生成流程（铁律）

### 1.1 日期确定（生成前必须执行）

```
步骤1：读取 env 中的 Today's date，确认为 T 日（如 2026-05-15）
步骤2：文件名日期 = T 日（如 20260515）
步骤3：搜索数据日期 = T - 1 日（如 2026-05-14）
步骤4：搜索关键词中使用 T-1 日期（如 "2026年5月14日 A股收盘"）
步骤5：header 显示 T 日日期，ticker 引用 T-1 日收盘数据
```

**特殊场景**：
| 场景 | header日期 | ticker引用数据 |
|------|-----------|---------------|
| 正常交易日 | T日 | T-1日收盘 |
| 周一 | T日 | 上周五（T-3）收盘 |
| 周末 | T日 | 最近交易日收盘，标注"周末休市" |
| 假期中 | T日 | 最近交易日收盘，标注"假期休市" |

### 1.2 4轮搜索（总计不超过24次）

| 轮次       | 必搜内容                                                             | 用于Tab   |
| -------- | ---------------------------------------------------------------- | ------- |
| **第1轮**  | A股四大指数+成交额+北向、美股三大指数+板块、国内外重大新闻、央行操作                             | 0/1/3/7 |
| **第2轮**  | 港股三大指数+个股、亚太（日经/KOSPI）+欧洲（DAX/CAC/富时）、大宗商品（WTI/黄金/白银）、加密货币、汇率+国债 | 3/4/6   |
| **第3轮A** | AI前沿（大模型/机器人/算力/应用）、13个关注标的基本面、论坛热门话题                            | 2/4/5/6 |
| **第3轮B** | 券商策略观点、板块估值（银行PB/股息率）、存款利率、理财收益率、保险产品                            | 4/6     |
| **第4轮**  | 估值参考表9行数据、长线资金动向、交叉验证关键数据                                        | 4/全部    |

**搜索原则**：

- 批量搜索：多个相关主题合并为一次搜索
- 优先权威来源：新华社、财联社、每经AI、新浪财经
- 搜索不到的数据：写"暂无数据"或引用最近可得日期并标注

### 1.3 撰写报告（基于template.html）

**操作步骤**：

1. 读取 template.html 作为基础模板
2. 复制模板内容到新文件 LaoshengDailyBrief_YYYYMMDD.html
3. 保留完整CSS、HTML骨架、switchTab函数、footer等等
4. 仅重写各tab-panel内部正文内容、header日期和ticker数据

### 1.4 更新index.html

- 在reports数组最前面添加新日期条目
- 历史记录只保留最近10天（不包含当天）
- index.html密码：`12345678`

## 二、数据引用规范（铁律）

### 2.1 数据时效性要求

| 数据类型    | 要求             | 示例                     |
| ------- | -------------- | ---------------------- |
| 价格/指数数据 | 必须引用最新可得日期     | "标普5/12收盘7412.84"      |
| 基本面数据   | 可引用最近可得，必须标注来源 | "截至2025年报"             |
| 政策/事件   | 必须引用最新进展       | "美联储5/7决议"             |
| 利率/国债   | 引用最新可得         | "1年定存1.65%（2026年5月数据）" |

### 2.2 禁止行为

- ❌ 严禁编造数据
- ❌ 严禁使用过期数据冒充当日数据
- ❌ 严禁"银行护盘"等无来源说法

### 2.3 数据标注格式

- 价格数据：`"标普5/12收盘7412.84（+0.19%）"`
- 基本面数据：`"截至2025年报：归母净利润3685.62亿元"`
- 未披露数据：`"一季报尚未单独披露（最新数据截至2025年报）"`

## 三、原创性规范（铁律）

### 3.1 禁止复制旧报告

**什么是"复制"**：将昨天的文字通过同义词替换、句式调整、语序变化等方式"改头换面"，但实质内容完全相同

**什么是"重新撰写"**：基于今天搜索到的新数据/新事件，写出全新的分析逻辑和结论

**具体要求**：

- Tab 4/5/6/7的所有文字必须基于当日市场环境重写
- 禁止连续两天使用相同或高度相似的bullet文字
- 基本面数据可引用，但必须标注来源日期

### 3.2 标题也必须不同

- 不仅内容要重新撰写，**每个卡片的标题也必须基于当日事件重新拟定**
- ❌ 昨天"银行股息率是定存的5倍"，今天"银行股息率达定存5倍" → 高度相似，不通过
- ✅ 今天"国常会基建加码+外贸+14.9%+银行分红季，高股息迎三重催化" → 基于新事件拟定，通过

### 3.3 低估值板块必须轮换标的

- **禁止连续两天写相同的两个标的**
- 每天从关注标的池（13个）中选择2个**当日最有催化/最有故事**的标的

### 3.4 分析角度必须切换

- 即使讨论同一个板块（如银行），每天的**分析切入角度**必须不同
- ❌ 连续两天都从"防御属性"角度分析银行
- ✅ 周一从"防御属性"切入 → 周二从"国常会基建加码带动融资需求"切入

## 五、CSS类名规范（必须使用模板定义的类名）

| 组件      | CSS类                                                                                                                                                   |
| ------- | ------------------------------------------------------------------------------------------------------------------------------------------------------ |
| 卡片容器    | `card`                                                                                                                                                 |
| 卡片标题/正文 | `card-title` / `card-body`                                                                                                                             |
| 子版块/子标题 | `sub-section` / `sub-title`                                                                                                                            |
| 区块标题    | `section-title`                                                                                                                                        |
| 标签      | `tag tag-*`（finance/tech/geo/policy/ai/chip/robot/app/energy）                                                                                          |
| 股票卡片    | `stock-card` / `stock-header` / `stock-info-row` / `stock-name` / `stock-code` / `stock-price-area` / `stock-price` / `stock-change` / `stock-bullets` |
| 市场区块    | `market-block` / `market-row`                                                                                                                          |
| 涨跌标识    | `up`（红涨）/ `down`（绿跌）/ `neutral`（灰持平）                                                                                                                   |

**涨跌颜色规则（A股惯例，严格执行）**：

- 🔴 **上涨/利好/正面** → 红色（`#f85149` / `var(--red)` / `class="up"`）
- 🟢 **下跌/利空/负面** → 绿色（`#3fb950` / `var(--green)` / `class="down"`）
- ⚪ **持平/中性** → 灰色（`class="neutral"`）
- 🔵 **中性信息/数据** → 蓝色（`#00d4ff` / `var(--accent)`）
- 🟡 **警示/关注** → 黄色（`#f0b429`）

**常见易错场景**：

- "大涨X%"、"创新高"、"领涨" → 必须用**红色**，不能用绿色
- "大跌X%"、"净卖出"、"暴跌" → 必须用**绿色**，不能用红色
- "偏悲观"、"恐慌"、"谨慎" → 必须用**绿色**（负面=绿色）
- "偏乐观"、"信心"、"看好" → 必须用**红色**（正面=红色）
- "外围市场上涨"相关卡片背景和文字 → 用**红色**系，不能用绿色系
- 涨跌幅数值（如+3.14%、-2.04%）→ 上涨红色、下跌绿色
- 标签（如"今日领涨"）→ 红色背景+红色文字
  | 提示条     | `alert-bar` / `hot-bar` / `good-bar`                                                                                                                   |
  | 高亮项/汇总卡 | `highlight-item` / `summary-card`                                                                                                                      |
  | 网格布局    | `grid-2` / `grid-3` / `grid-4` / `grid-auto`                                                                                                           |
  | 特殊组件    | `insurance-flow` / `insurance-card` / `rate-compare` / `big-num` / `topic-card` / `institute-card` / `sentiment-item` / `timeline-item`                |

**禁止**：自定义CSS类名、混用Tailwind类（如`w-`、`h-`、`p-`、`flex`、`grid`等）

## 八、手机端适配

### 8.1 适配要求

- 所有tab需要适配手机展示（重要）
- Tab导航右侧渐变遮罩提示
- 表格`font-size:12px !important`，允许换行
- stock-info-row横向flex排列
- footer底部安全区适配

### 8.2 响应式断点

```css
/* 768px以下：平板/大屏手机 */
@media (max-width: 768px) {
  - 所有网格改为单列布局
  - 卡片内边距调整为14-16px
  - 字体大小适当缩小
  - 图表左侧padding减小
}

/* 480px以下：小屏手机 */
@media (max-width: 480px) {
  - 4/6列网格改为2列
  - 情绪指标改为2列
  - 字体进一步缩小
}
```

### 8.3 各Tab手机端适配要点

| Tab     | 关键适配                 |
| ------- | -------------------- |
| 0-要点速览  | 4张汇总卡片单列、8条要点单列      |
| 1-国内外新闻 | 新闻卡片单列、标签换行          |
| 2-AI前沿  | AI卡片单列、分类标签横向滚动      |
| 3-全球市场  | 市场数据卡片单列、表格横向滚动      |
| 4-价值投资  | 机构观点卡片单列、情绪指标3→2列    |
| 5-关注标的  | 13只股票卡片单列            |
| 6-理财话题  | 保险流程纵向、话题卡片单列、图表适配   |
| 7-今日总结  | 宏观/市场/资金面卡片单列、操作建议单列 |

### 8.4 新增组件手机端规范

**新增卡片/图表/网格时，必须同时添加手机端适配样式：**

```css
/* 示例：新增网格 */
.my-new-grid {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  gap: 16px;
}

/* 必须同时添加手机端适配 */
@media (max-width: 768px) {
  .my-new-grid {
    grid-template-columns: 1fr;
    gap: 12px;
  }
}
```

**关键原则：**

- 所有`grid-template-columns`必须在手机端改为单列或双列
- 所有卡片内边距在手机端减小至14-16px
- 所有大号字体（>20px）在手机端适当缩小
- 所有图表左侧留白在手机端减小（如120px→80px）
