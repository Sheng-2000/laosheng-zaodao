## 一、生成流程（铁律）

### 1.1 日期确定（生成前必须执行）

```
步骤1：读取 env 中的 Today's date，确认为 T 日（如 2026-05-15）
步骤2：文件名日期 = T 日（如 20260515）
步骤3：搜索数据日期 = T - 1 日（如 2026-05-14）
步骤4：搜索关键词中使用 T-1 日期（如 "2026年5月14日 A股收盘"）
步骤5：header 显示 T 日日期，ticker 引用 T-1 日收盘数据
```

**特殊场景**：
| 场景 | header日期 | ticker引用数据 |
|------|-----------|---------------|
| 正常交易日 | T日 | T-1日收盘 |
| 周一 | T日 | 上周五（T-3）收盘 |
| 周末 | T日 | 最近交易日收盘，标注"周末休市" |
| 假期中 | T日 | 最近交易日收盘，标注"假期休市" |

### 1.2 4轮搜索（总计不超过24次）

| 轮次       | 必搜内容                                                             | 用于Tab   |
| -------- | ---------------------------------------------------------------- | ------- |
| **第1轮**  | A股四大指数+成交额+北向、美股三大指数+板块、国内外重大新闻、央行操作                             | 0/1/3/7 |
| **第2轮**  | 港股三大指数+个股、亚太（日经/KOSPI）+欧洲（DAX/CAC/富时）、大宗商品（WTI/黄金/白银）、加密货币、汇率+国债 | 3/4/6   |
| **第3轮A** | AI前沿（大模型/机器人/算力/应用）、13个关注标的基本面、论坛热门话题                            | 2/4/5/6 |
| **第3轮B** | 券商策略观点、板块估值（银行PB/股息率）、存款利率、理财收益率、保险产品                            | 4/6     |
| **第4轮**  | 估值参考表9行数据、长线资金动向、交叉验证关键数据                                        | 4/全部    |

**搜索原则**：

- 批量搜索：多个相关主题合并为一次搜索
- 优先权威来源：新华社、财联社、每经AI、新浪财经
- 搜索不到的数据：写"暂无数据"或引用最近可得日期并标注

### 1.3 撰写报告（基于template.html）

**操作步骤**：

1. 读取 template.html 作为基础模板
2. 复制模板内容到新文件 LaoshengDailyBrief_YYYYMMDD.html
3. 根据 搜索到的信息 填写 {{...}}、{...} 占位符，填写文字需要丰富一些（数据除外）
4. 所有的tab页面中文字内容中重点 需要突出展示，注意字体大小不要改变检查 所有的tab
5. 所有的tab页面都需要遵循 涨红跌绿的展示要求

- 历史记录只保留最近10天（不包含当天）

## 二、数据引用规范（铁律）

### 2.1 数据时效性要求

| 数据类型    | 要求             | 示例                     |
| ------- | -------------- | ---------------------- |
| 价格/指数数据 | 必须引用最新可得日期     | "标普5/12收盘7412.84"      |
| 基本面数据   | 可引用最近可得，必须标注来源 | "截至2025年报"             |
| 政策/事件   | 必须引用最新进展       | "美联储5/7决议"             |
| 利率/国债   | 引用最新可得         | "1年定存1.65%（2026年5月数据）" |

### 2.2 禁止行为

- ❌ 严禁编造数据
- ❌ 严禁使用过期数据冒充当日数据
- ❌ 严禁"银行护盘"等无来源说法

### 2.3 数据标注格式

- 价格数据：`"标普5/12收盘7412.84（+0.19%）"`
- 基本面数据：`"截至2025年报：归母净利润3685.62亿元"`
- 未披露数据：`"一季报尚未单独披露（最新数据截至2025年报）"`

**涨跌颜色规则（A股惯例，严格执行）**：

- 🔴 **上涨/利好/正面** → 红色（`#f85149` / `var(--red)` / `class="up"`）
- 🟢 **下跌/利空/负面** → 绿色（`#3fb950` / `var(--green)` / `class="down"`）
- ⚪ **持平/中性** → 灰色（`class="neutral"`）
- 🔵 **中性信息/数据** → 蓝色（`#00d4ff` / `var(--accent)`）
- 🟡 **警示/关注** → 黄色（`#f0b429`）

**常见易错场景**：

- "大涨X%"、"创新高"、"领涨" → 必须用**红色**，不能用绿色
- "大跌X%"、"净卖出"、"暴跌" → 必须用**绿色**，不能用红色
- "偏悲观"、"恐慌"、"谨慎" → 必须用**绿色**（负面=绿色）
- "偏乐观"、"信心"、"看好" → 必须用**红色**（正面=红色）
- "外围市场上涨"相关卡片背景和文字 → 用**红色**系，不能用绿色系
- 涨跌幅数值（如+3.14%、-2.04%）→ 上涨红色、下跌绿色
- 标签（如"今日领涨"）→ 红色背景+红色文字

## 四、格式规范

- **涨红跌绿**：`.up`=红色（涨），`.down`=绿色（跌），`.neutral`=灰色（持平）
- **搜索不到的数据**：标注"暂无数据"，不编造
- **零"待更新"为目标**：亚太5个+大宗4个+汇率4个尽量全部搜齐
- 所有的文字内容需要突出重点（通过颜色突出，不要改字体大小和其他的）
