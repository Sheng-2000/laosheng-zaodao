# 老盛早知道报告质量检查清单（优化版）

> **核心原则**：先查数据，再查结构，最后查细节。不通过即打回，不得跳过。

---

## 一、数据及时性检查（⭐⭐⭐ 最高优先级）

**检查时机**：所有其他检查之前必须首先执行
**不通过处理**：发现旧数据立即打回，重新搜索替换

### 1.1 关键日期定义

- **报告日期** = T日（如5月21日）
- **数据日期** = T-1日（如5月20日）
- **模板旧日期** = template.html中残留的日期

### 1.2 自动化检测（必运行）

**检测步骤**：
1. 检查旧日期残留（T-3日及更早，输出应≤3处）
2. 验证新数据已写入（关键数据出现≥10处）
3. 检查数据一致性（同一指标在不同位置应相同）

### 1.3 判定标准

| 检查项   | 通过标准           | 不通过信号   |
| ----- | -------------- | ------- |
| 旧日期残留 | ≤3处（仅允许合理历史回顾） | >3处     |
| 新数据覆盖 | ≥10处关键数据       | <5处     |
| 数据一致性 | 同一指标全报告一致      | 同一指标不同值 |

### 1.4 逐Tab抽查清单

| Tab   | 抽查位置           | 检查内容        |
| ----- | -------------- | ----------- |
| Tab 0 | 4张摘要卡片数据       | 是否为T-1日数据   |
| Tab 1 | 第1张新闻卡片        | 是否为最近1-2天新闻 |
| Tab 3 | A股market-block | 指数是否为T-1日收盘 |
| Tab 5 | 工商银行stock-card | 股价是否为T-1日   |
| Tab 7 | 市场面card-body   | 描述是否为T-1日市场 |

## 三、数据准确性检查

### 3.1 涨跌颜色检查

- [ ] A股/港股/美股：**涨红跌绿**（.up红色，.down绿色）
- [ ] 加密货币/外汇：涨红跌绿（国际惯例）

### 3.2 数据标注检查

- [ ] 价格/指数数据标注日期（如"5/20收盘"）
- [ ] 无旧日期残留（引用前日数据除外）

### 3.3 数据缺失检查（⭐新增）

**问题描述**：股价、指数等关键数据可能显示为"--"或缺失。

**检查方法**：

- [ ] 检查是否有 "--" 缺失数据标记
- [ ] 验证建设银行股价已填充
- [ ] 验证招商银行股价已填充
- [ ] 验证农业银行股价已填充
- [ ] 验证大秦铁路股价已填充
- [ ] 验证宁波银行股价已填充

**常见缺失位置**：

| 位置       | 数据项                    | 说明       |
| -------- | ---------------------- | -------- |
| 页头ticker | 深证/创业板/道指              | 页头指数数据   |
| 亚太市场     | KOSPI/台湾/印度/澳洲         | 可标注为"震荡" |
| 大宗商品     | 国际黄金/白银                | 美元/盎司价格  |
| 加密货币     | 以太坊                    | 美元价格     |
| 估值参考     | 上证PE/中海油桶油利润/移动PB/核电PE | 估值数据     |
| 股票卡片     | 建行/招行/农行/大秦/宁波银行       | 具体股价     |

**合格标准**：缺失数据数量 = 0

## 五、原创性检查

### 5.1 检查方法

将当天报告Tab 4/5/6/7与前一天报告逐段对比：

| 检查项            | 通过标准       |
| -------------- | ---------- |
| 卡片标题           | 基于当日事件重新拟定 |
| Tab 5 bullet文字 | 基于最新市场环境撰写 |
| Tab 6/7文字      | 基于当日数据撰写   |
| 低估值板块          | 2个标的与前一天不同 |

### 5.2 快速判定

- [ ] 无>10字连续相同
- [ ] 无同义词替换+分析逻辑完全相同
- [ ] 分析角度与前一天不同

## 六、常见错误与红线（重点提醒）

### 6.1 🔴 极高优先级错误

| 错误                  | 后果      | 检查方式                      |
| ------------------- | ------- | ------------------------- |
| 子代理直接创建HTML文件       | CSS完全丢失 | 文件行数<2700                 |
| 模板数据未完全替换           | 整份报告过时  | grep旧日期>3处                |
| 混用Tailwind类名        | 样式污染    | grep -oE 'class="[^"]*w-' |
| switchTab缺少scrollTo | 交互失效    | grep -c "window.scrollTo" |

### 6.2 🔴 高优先级错误

| 错误             | 后果        | 检查方式                             |
| -------------- | --------- | -------------------------------- |
| 数据不一致          | 同一指标不同值   | grep同一指标                         |
| 遗漏债券基金         | 结构不完整     | grep -c "债券基金"                   |
| 遗漏今日操作建议       | 结构不完整     | grep -c "今日操作建议"                 |
| **社区情绪不完整**    | **结构不完整** | **grep -c "sentiment-item" ≠ 9** |
| **Tab 6 内容重复** | **模板残留**  | **内容匹配检查**                       |

#### 社区情绪sentiment-item检查（⭐重要）

**固定顺序（9个，必须完整）**：

```bash
# 检查sentiment-item数量必须为9个
grep -c 'class="sentiment-item"' 老盛早知道_YYYYMMDD.html

# 检查9个固定顺序（高股息→银行→公用事业→电力→AI算力→能源油价→新能源→黄金→加密货币）
grep -o 'sentiment-name">[^<]*' 老盛早知道_YYYYMMDD.html | head -9
```

**9个固定顺序**：

1. 💰 高股息
2. 🏦 银行  
3. ⚡ 公用事业
4. 🔌 电力
5. 🤖 AI算力
6. 🛢️ 能源油价
7. 🌱 新能源
8. 🥇 黄金
9. ₿ 加密货币

**合格标准**：

- 数量 = 9个
- 顺序 = 固定顺序（如上）
- 每个包含：emoji图标 + sentiment-name + progress-bar + percent + desc

#### Tab 6 内容匹配检查（⭐新增）

**问题描述**：Tab 6 中多个子章节容易出现模板内容重复，即不同标题下显示相同内容。

**检查方法**：

- [ ] 家庭保险配置内容正确（医疗险/重疾险/意外险/定期寿险，不含理财工具对比进度条）
- [ ] 黄金投资内容正确（黄金价格走势+四种投资方式，不含股息率对比）
- [ ] 债券基金内容正确（纯债/一级/二级/可转债类型介绍，不含理财工具进度条）

**正确内容对应**：

| 子章节    | 正确内容                     | 错误内容（模板残留）     |
| ------ | ------------------------ | -------------- |
| 家庭保险配置 | 医疗险/重疾险/意外险/定期寿险 + 分红险提醒 | 稳健理财工具对比进度条    |
| 黄金投资   | 黄金价格走势 + 四种投资方式          | 股息率对比 + 资产配置建议 |
| 债券基金   | 纯债/一级/二级/可转债类型介绍         | 稳健理财工具对比进度条    |

### 6.3 子代理使用规范

| ❌ 错误做法        | ✅ 正确做法               |
| ------------- | -------------------- |
| 子代理直接创建HTML文件 | 只让子代理输出文本，主进程负责文件操作  |
| 子代理修改模板       | 只让子代理读取模板，主进程负责复制和修改 |
| 子代理返回完整HTML   | 子代理只返回各Tab片段，主进程负责组装 |
