# 老盛早知道报告质量检查清单（优化版）

> **核心原则**：先查数据，再查结构，最后查细节。不通过即打回，不得跳过。

---

## 一、数据及时性检查（⭐⭐⭐ 最高优先级）

**检查时机**：所有其他检查之前必须首先执行
**不通过处理**：发现旧数据立即打回，重新搜索替换

### 1.1 关键日期定义

- **报告日期** = T日（如5月21日）
- **数据日期** = T-1日（如5月20日）
- **模板旧日期** = template.html中残留的日期

### 1.2 自动化检测（必运行）

```bash
# 1. 检查旧日期残留（T-3日及更早，输出应≤3处）
grep -c "5月1[2-6]日\|5/1[2-6]" 老盛早知道_YYYYMMDD.html

# 2. 验证新数据已写入（关键数据出现≥10处）
grep -c "4162\.18\|科创50.*3\.2\|中美.*300亿" 老盛早知道_YYYYMMDD.html

# 3. 检查数据一致性（同一指标在不同位置应相同）
grep -n "科创50\|恒生指数\|黄金" 老盛早知道_YYYYMMDD.html | head -20
```

### 1.3 判定标准

| 检查项   | 通过标准           | 不通过信号   |
| ----- | -------------- | ------- |
| 旧日期残留 | ≤3处（仅允许合理历史回顾） | >3处     |
| 新数据覆盖 | ≥10处关键数据       | <5处     |
| 数据一致性 | 同一指标全报告一致      | 同一指标不同值 |

### 1.4 逐Tab抽查清单

| Tab   | 抽查位置           | 检查内容        |
| ----- | -------------- | ----------- |
| Tab 0 | 4张摘要卡片数据       | 是否为T-1日数据   |
| Tab 1 | 第1张新闻卡片        | 是否为最近1-2天新闻 |
| Tab 3 | A股market-block | 指数是否为T-1日收盘 |
| Tab 5 | 工商银行stock-card | 股价是否为T-1日   |
| Tab 7 | 市场面card-body   | 描述是否为T-1日市场 |

## 二、结构与样式检查（⭐核心检查）

### 2.1 文件完整性（自动化检查）

```bash
# 1. 文件行数检查（目标3300±100行）
wc -l 老盛早知道_YYYYMMDD.html

# 2. CSS完整性检查（</style>应在文件末尾附近）
grep -n '</style>' 老盛早知道_YYYYMMDD.html

# 3. 结构元素数量对比（必须与模板一致）
for cls in "sub-section" "stock-card" "grid-4" "grid-2" "grid-3" "insurance-flow" "insurance-card" "rate-compare" "big-num" "topic-card" "tab-panel" "sub-title" "section-title"; do
  tpl_cnt=$(grep -c "class=\"$cls\"" template.html 2>/dev/null || echo "0")
  rpt_cnt=$(grep -c "class=\"$cls\"" 老盛早知道_YYYYMMDD.html 2>/dev/null || echo "0")
  if [ "$tpl_cnt" = "$rpt_cnt" ]; then echo "✅ $cls: $rpt_cnt"; else echo "❌ $cls: 模板=$tpl_cnt 报告=$rpt_cnt"; fi
done

# 4. CSS类名一致性（输出必须为空）
grep -oP 'class="[^"]*"' template.html | sort -u > /tmp/tpl.txt
grep -oP 'class="[^"]*"' 老盛早知道_YYYYMMDD.html | sort -u > /tmp/rpt.txt
comm -23 /tmp/rpt.txt /tmp/tpl.txt  # 报告多出的类
comm -13 /tmp/rpt.txt /tmp/tpl.txt  # 报告缺失的类

# 5. 易遗漏项检查
echo "=== 易遗漏项 ==="
echo "关注标的深度解读: $(grep -c '关注标的深度解读' 老盛早知道_YYYYMMDD.html)"
echo "债券基金: $(grep -c '债券基金' 老盛早知道_YYYYMMDD.html)"
echo "国内黄金: $(grep -c '国内黄金' 老盛早知道_YYYYMMDD.html)"
echo "中国10年期国债: $(grep -c '中国10年期国债' 老盛早知道_YYYYMMDD.html)"
echo "美国10年期国债: $(grep -c '美国10年期国债' 老盛早知道_YYYYMMDD.html)"
echo "社区情绪sentiment-item: $(grep -c 'sentiment-item' 老盛早知道_YYYYMMDD.html) (必须为9个)"

# 6. 社区情绪sentiment-item详细检查
echo "=== 社区情绪sentiment-item检查 ==="
sentiment_count=$(grep -c 'class="sentiment-item"' 老盛早知道_YYYYMMDD.html)
echo "sentiment-item数量: $sentiment_count (必须为9个)"
if [ "$sentiment_count" -eq 9 ]; then
    echo "✅ 数量正确"
    echo "9个固定顺序:"
    grep -o 'sentiment-name">[^<]*' 老盛早知道_YYYYMMDD.html | head -9 | nl
else
    echo "❌ 数量错误，应为9个"
fi

# 7. Tailwind污染检查（输出必须为空）
grep -oE 'class="[^"]*(w-|h-|p-|m-|flex|grid|block|text-|bg-|rounded)' 老盛早知道_YYYYMMDD.html | head -5
```

### 2.2 快速通过标准

| 检查项        | 通过标准                | 不通过信号          |
| ---------- | ------------------- | -------------- |
| 文件行数       | 3200±300行           | <2900或>3500    |
| CSS位置      | </style>在1300-1500行 | <1000行（CSS被截断） |
| tab-panel  | 8个                  | 数量不符           |
| sub-title  | 25-26个              | 数量不符           |
| CSS类差异     | 输出为空                | 有缺失/多出         |
| Tailwind污染 | 输出为空                | 混用Tailwind类    |

## 五、原创性检查

### 5.1 检查方法

将当天报告Tab 4/5/6/7与前一天报告逐段对比：

| 检查项            | 通过标准       |
| -------------- | ---------- |
| 卡片标题           | 基于当日事件重新拟定 |
| Tab 5 bullet文字 | 基于最新市场环境撰写 |
| Tab 6/7文字      | 基于当日数据撰写   |
| 低估值板块          | 2个标的与前一天不同 |

### 5.2 快速判定

- [ ] 无>10字连续相同
- [ ] 无同义词替换+分析逻辑完全相同
- [ ] 分析角度与前一天不同

## 六、常见错误与红线（重点提醒）

### 6.1 🔴 极高优先级错误

| 错误                  | 后果      | 检查方式                      |
| ------------------- | ------- | ------------------------- |
| 子代理直接创建HTML文件       | CSS完全丢失 | 文件行数<3000                 |
| 模板数据未完全替换           | 整份报告过时  | grep旧日期>3处                |
| 混用Tailwind类名        | 样式污染    | grep -oE 'class="[^"]*w-' |
| switchTab缺少scrollTo | 交互失效    | grep -c "window.scrollTo" |

### 6.2 🔴 高优先级错误

| 错误       | 后果      | 检查方式             |
| -------- | ------- | ---------------- |
| 数据不一致    | 同一指标不同值 | grep同一指标         |
| 遗漏债券基金   | 结构不完整   | grep -c "债券基金"   |
| 遗漏今日操作建议 | 结构不完整   | grep -c "今日操作建议" |
| **社区情绪不完整** | **结构不完整** | **grep -c "sentiment-item" ≠ 9** |

#### 社区情绪sentiment-item检查（⭐重要）

**固定顺序（9个，必须完整）**：

```bash
# 检查sentiment-item数量必须为9个
grep -c 'class="sentiment-item"' 老盛早知道_YYYYMMDD.html

# 检查9个固定顺序（高股息→银行→公用事业→电力→AI算力→能源油价→新能源→黄金→加密货币）
grep -o 'sentiment-name">[^<]*' 老盛早知道_YYYYMMDD.html | head -9
```

**9个固定顺序**：
1. 💰 高股息
2. 🏦 银行  
3. ⚡ 公用事业
4. 🔌 电力
5. 🤖 AI算力
6. 🛢️ 能源油价
7. 🌱 新能源
8. 🥇 黄金
9. ₿ 加密货币

**合格标准**：
- 数量 = 9个
- 顺序 = 固定顺序（如上）
- 每个包含：emoji图标 + sentiment-name + progress-bar + percent + desc

### 6.3 子代理使用规范

| ❌ 错误做法        | ✅ 正确做法               |
| ------------- | -------------------- |
| 子代理直接创建HTML文件 | 只让子代理输出文本，主进程负责文件操作  |
| 子代理修改模板       | 只让子代理读取模板，主进程负责复制和修改 |
| 子代理返回完整HTML   | 子代理只返回各Tab片段，主进程负责组装 |
