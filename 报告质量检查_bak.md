# 老盛早知道报告质量检查清单

> 每次生成报告后，必须按照此清单逐项检查，确保与 template.html 结构完全一致

## 零、数据及时性检查（⭐⭐⭐ 最高优先级，必须首先执行）

> **核心问题**：template.html 是基于旧日期的完整报告，生成时极易遗漏替换旧数据。此检查必须在所有其他检查之前执行，不通过则直接打回重做。

### 0.1 检查方法

**步骤1：确定关键日期**

- 报告日期 = T日（如5月21日）
- 搜索数据日期 = T-1日（如5月20日）
- 模板旧日期 = template.html 中残留的日期（如5月13日等）

**步骤2：逐Tab检查数据是否为T-1日最新数据**

对每个Tab面板（panel-0 到 panel-7），逐一检查：

| Tab         | 重点检查内容                              | 合格标准                |
| ----------- | ----------------------------------- | ------------------- |
| Tab 0 要点速览  | 4张摘要卡片的数据、8条亮点的事件描述、时间线日历           | 数据为T-1日，事件为最近发生     |
| Tab 1 国内外新闻 | 12张新闻卡片的事件描述                        | 新闻为T-1日或最近1-2天的真实新闻 |
| Tab 2 AI前沿  | 大模型/机器人/算力等卡片内容                     | 描述最新的AI行业动态         |
| Tab 3 全球市场  | A股/港股/美股/欧股/亚太/大宗商品/加密货币/汇率债券的指数和价格 | 所有数据为T-1日收盘数据       |
| Tab 4 价值投资  | 机构观点、社区情绪、高股息分析                     | 基于T-1日市场环境撰写        |
| Tab 5 关注标的  | 13个stock-card的价格、涨跌幅、bullet文字       | 价格和涨跌幅为T-1日数据       |
| Tab 6 理财话题  | 定存利率、股息率、黄金价格、国债收益率                 | 利率为最新公布值，价格为T-1日    |
| Tab 7 今日总结  | 关键词、宏观面、市场面、资金面、操作建议                | 基于T-1日数据撰写          |

**步骤3：自动化检测命令**

```bash
# 1. 提取template.html中的旧数据特征值（替换为实际旧值）
# 检查旧指数/价格是否残留（必须输出为空或极少）
grep -n "旧上证指数\|旧标普\|旧黄金价格" 老盛早知道_YYYYMMDD.html

# 2. 检查旧日期残留（T-3日及更早的日期不应出现，除非是合理历史回顾）
grep -c "T-3日日期\|T-4日日期" 老盛早知道_YYYYMMDD.html

# 3. 验证T-1日新数据已写入
grep -c "T-1日上证指数\|T-1日标普\|T-1日黄金价格" 老盛早知道_YYYYMMDD.html
```

### 0.2 判定标准

| 检查项        | 通过标准             | 不通过信号        |
| ---------- | ---------------- | ------------ |
| Tab 0 数据   | 摘要卡片数据为T-1日      | 数据为T-3日或更早   |
| Tab 1 新闻   | 新闻为最近1天          | 新闻为一周前或更早    |
| Tab 3 市场数据 | 指数/价格为T-1日收盘     | 指数/价格与T-1日不符 |
| Tab 5 标的价格 | 股价为T-1日收盘价       | 股价为旧日期数据     |
| 旧日期残留      | ≤ 3处（仅允许合理历史回顾）  | > 3处旧日期      |
| 新数据覆盖      | T-1日关键数据出现 ≥ 10处 | < 5处         |

### 0.3 不通过处理

- **发现任何Tab存在旧数据** → 标记具体位置 → 重新搜索最新数据 → 替换旧数据 → 重新检查
- **发现模板旧内容未替换** → 重新生成该Tab的全部内容
- **多次不通过** → 检查生成流程是否存在系统性问题（如直接复制模板未替换）

### 0.4 常见数据过时场景

| 场景       | 表现                           | 根本原因                       |
| -------- | ---------------------------- | -------------------------- |
| 整份报告数据过时 | 所有Tab的数据都是几天前的               | 生成时直接复制了template.html未替换内容 |
| 部分Tab过时  | Tab 3数据最新但Tab 0/1/4/5/7仍是旧数据 | 只更新了市场数据Tab，未更新其他Tab       |
| 新闻过时     | Tab 1的新闻是几天前的旧闻              | 未搜索最新新闻                    |
| 标的价格过时   | Tab 5的股价与实际不符                | 未搜索最新股价数据                  |
| 分析文字过时   | Tab 4/7的分析引用的是旧事件            | 分析文字基于旧市场环境撰写              |

## 九、内容重写验证检查（⭐⭐⭐ 最高优先级）

> **核心问题**：template.html 是一份基于旧日期的完整报告，不是空白模板。
> 如果只替换了 Header Ticker 和 Tab 3 的数字，而 Tab 0/1/2/4/5/6/7 的正文内容仍然是模板旧内容，则报告不合格。

### 9.1 通用检查方法（适用于任何日期）

**步骤1：确定关键日期**

- 报告日期 = T日（如5月15日）
- 搜索数据日期 = T-1日（如5月14日）
- 模板旧日期 = T-2日（如5月13日，因为模板是前一天生成的）

**步骤2：检查模板旧日期残留**

```bash
# 检查T-2日日期格式是否残留（必须输出为空或极少）
# 将MM/DD替换为实际的T-2日日期
grep -n "5月13日\|5/13" 老盛早知道_YYYYMMDD.html  # 示例

# 检查T-2日指数数据是否残留（从template.html中提取）
grep -n "4214\.49\|4225\.02" 老盛早知道_YYYYMMDD.html  # 示例
```

**步骤3：验证新数据已写入**

```bash
# 检查T-1日数据是否存在（从搜索结果中提取关键数据）
grep -n "4177\.92\|50063\.46" 老盛早知道_YYYYMMDD.html  # 示例
```
